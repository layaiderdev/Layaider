New to aider? Start here
Layaider drives aider (the AI pair-programmer). You assemble small reusable building blocks, then a blueprint ties them together for a run:
  1. Key — your model provider's API key (stored on-device at ~/.layaider, mode 0600; never committed).
  2. Model — a model string, e.g. openrouter/google/gemma-3-27b-it:free or anthropic/claude-....
  3. Blueprint — pick a model for the architect role and attach the matching key. Editor/weak roles are optional; start with just the architect.
  4. Launch — open the Live tab and start the blueprint; the session runs in the background and survives tab switches.
For a fully offline setup (Ollama and other local servers), see docs/LOCAL_MODELS.md in the repo.

About the Live terminal
Drive a running aider session here. Launch a blueprint to start one (it runs in tmux and survives tab switches); then type messages and send with Enter. The ⋯ menu opens the command palette and session controls; A−/A+ change the font and ⛶ is full-screen (Esc exits). When aider asks a yes/no question, answer buttons appear above the input.
